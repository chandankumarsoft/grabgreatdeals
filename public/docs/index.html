Overview

Base URL
http://localhost:8080/api/v1
Format
JSON
Authentication
JWT Bearer
API Version
v1

Response Envelope

Every response — success or error — uses the same wrapper:

{
  "status": true,
  "message": "Human-readable message",
  "data": { },
  "errors": []
}
FieldTypeDescription
statusbooleantrue on success, false on error
messagestringHuman-readable summary
dataobject | array | nullResponse payload
errorsarrayValidation or error detail

HTTP Status Codes

CodeMeaning
200OK
201Created
400Bad Request
401Unauthorized — missing or invalid token
403Forbidden — valid token, insufficient role
404Not Found
422Unprocessable Entity — validation failed
500Internal Server Error

Authentication

Pass the JWT token in the Authorization header: 

Authorization: Bearer <token>

Tokens are returned by the Login and Register endpoints. The token contains sub (user ID), email, role, and expires according to server config.

18. Webhooks

POST /webhooks/payment/{gateway}
public
▼

Receives payment status notifications from external gateways. Secured by payload validation — no JWT required.

URL — gateway values

fpx  |  card  |  ewallet  |  bank_transfer  |  cod

Request Body
FieldTypeRequiredDescription
order_numberstringYesGGD order number e.g. GGD-AB12CD34-20260407
statusstringYessuccess | failed | refunded
transaction_idstringNoGateway transaction reference
gateway_responsestringNoRaw gateway response (JSON string)
Example Request
{
  "order_number": "GGD-AB12CD34-20260407",
  "status": "success",
  "transaction_id": "FPX-TXN-001",
  "gateway_response": "{\"code\":\"00\",\"message\":\"Success\"}"
